# Local Danmu Player

一个浏览器本地视频播放器体验项目：视频文件不上传，浏览器计算文件前 16 MiB 的 MD5，并通过服务端安全代理调用弹弹play开放弹幕网络完成匹配和弹幕加载。

## 功能

- 拖放或选择本地视频播放
- 使用文件名、文件大小、视频时长和前 16 MiB Hash 自动匹配
- 手动搜索作品和剧集
- 弹幕滚动、顶部/底部弹幕显示
- 弹幕开关、透明度、字号、速度设置
- 可配置的 `+N 秒` 跳过片头按钮，默认 80 秒
- 播放进度和设置保存到浏览器本地
- 弹幕缓存保存到 IndexedDB，减少重复请求
- 未配置 API 凭证时使用明确标注的演示弹幕

## 本地运行

```powershell
npm install
Copy-Item .env.example .env
npm run dev
```

浏览器打开 `http://localhost:5173`。没有凭证时仍可体验播放器、拖放、跳转、弹幕层和本地存储；审核通过后把凭证写入 `.env`，重启服务即可启用真实接口。

## 生产运行

```powershell
npm run build
node server/index.mjs
```

生产环境需要设置：

- `DANDANPLAY_APP_ID`
- `DANDANPLAY_APP_SECRET`
- `PORT`（可选，默认 `8787`）

生产部署时将 `dist` 和 `server` 一起部署，AppSecret 只放服务端环境变量，不提交到仓库、不写入网页代码。

## API 使用说明

项目只在用户主动选择视频或执行手动搜索后请求接口：

- `POST /api/v2/match`
- `GET /api/v2/search/episodes`
- `GET /api/v2/comment/{episodeId}`

服务端按照官方文档使用 `X-AppId`、`X-Timestamp`、`X-Signature` 请求头签名。缓存默认有效 6 小时，避免重复获取弹幕。

## 合规边界

- 不上传或转存用户的视频文件
- 不批量抓取弹幕或搜索结果
- 不对弹幕数据进行售卖或二次分发
- 尊重接口缓存、调用和安全要求
- 页面明确展示数据来源和演示模式状态
